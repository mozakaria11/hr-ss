# نظام سواعد المتكامل — إصدار الخادم 1.10

نسخة تشغيل فعلية قابلة للتركيب على cPanel أو VPS أو Docker. تم تحويل النموذج التفاعلي إلى نظام يعتمد جلسات دخول آمنة وقاعدة بيانات ومزامنة مركزية وسجل تدقيق وواجهات API ورفع مستندات.

## ما يعمل في هذا الإصدار

- تسجيل الدخول والخروج، كلمات مرور مشفرة، حماية CSRF، جلسات آمنة، وأدوار النظام الثمانية.
- دخول كل موظف برقم الهوية/الإقامة أو البريد الإلكتروني، مع قفل مؤقت بعد تكرار المحاولات الفاشلة.
- استعادة كلمة المرور بالبريد عبر رابط أحادي الاستخدام صالح لمدة 30 دقيقة دون كشف وجود الحساب.
- إنشاء الموظف والعقد وحساب الدخول وربطها داخل معاملة واحدة، مع صورة شخصية وبيانات الهوية والجوال والجنسية واللغة.
- صلاحيات مستقلة لإضافة الموظف وتعديل بياناته وإدارة حساب دخوله، مع منحها افتراضيًا للمدير العام والمدير التنفيذي ومدير الموارد البشرية.
- تعديل ملف الموظف وبيانات الهوية والتواصل والتوظيف والراتب من شاشة الموارد البشرية مع تحديث الحساب والعقد المرتبطين.
- وضع موظف محسّن للجوال ومشروع Android WebView آمن يفتح النظام من النطاق الرسمي ويدعم الموقع والكاميرا ورفع الملفات.
- لوحة القيادة والوحدات الموجودة في النموذج: المشاريع، الموارد البشرية، الحضور، الرواتب، الحسابات، العملاء، العقود، المشتريات، الموردون، مقاولو الباطن، المستودع، المعدات، الصيانة، الجودة، السلامة، الوثائق، الموافقات، الضرائب، التقارير، البوابات والإعدادات.
- حفظ بيانات التشغيل المركزية في MySQL بدل ضياعها عند إغلاق المتصفح.
- REST API منظم للمشاريع والموظفين والمعدات والمخزون والموردين والموافقات.
- سجل تدقيق للدخول والحفظ والإنشاء والتعديل والحذف والرفع.
- رفع PDF/JPG/PNG/DOCX/XLSX حتى 20 MB إلى مجلد غير متاح مباشرة من الويب.
- مُثبّت ويب، فحص صحة، Docker، ومخططات MySQL وSQLite.
- مواصفات نقل البيانات والإقفال المالي ودليل التدريب المرفقة مع النسخة السابقة.
- حضور وانصراف فعلي من تطبيق الموظف عبر GPS ونطاقات المشاريع، مع دعم ورديتي النهار والليل وفترة سماح 15 دقيقة.
- لوحة حضور مباشرة، وإقفال غياب الأيام السابقة مع استثناء الإجازات المعتمدة ويوم الجمعة.
- مسير رواتب شهري متعدد المراحل: احتساب، مراجعة، اعتماد، ترحيل، ثم إنشاء قيد محاسبي متوازن تلقائيًا.
- ربط حساب دخول المستخدم بملف موظف واحد، وإظهار سجل الحضور وآخر قسيمة راتب مرحّلة في تطبيق الموظف.
- دورة مواد ومشتريات فعلية: طلب واعتماد حسب سقف الصلاحية، أمر شراء لمورد مؤهل، استلام وفحص ثم إضافة الرصيد.
- مستودعات متعددة، أرصدة مستقلة، سجل حركات، صرف للمشاريع، ومنع الرصيد السالب على مستوى الخادم.
- سجل معدات فعلي ومحاضر نقل بين المشاريع لا تغيّر التوزيع إلا بعد اعتماد مدير المشاريع أو المدير العام.
- عقود عملاء ومقاولي باطن، ومستخلصات تراكمية لا تتجاوز قيمة العقد.
- احتساب المحتجز واسترداد الدفعة والخصومات وضريبة القيمة المضافة تلقائيًا.
- إصدار فواتير داخلية وقيود إيراد، وتسجيل تحصيلات جزئية أو كاملة مع قيود متوازنة.
- لوحة ربحية مشروع تجمع الإيراد المفوتر والتحصيل والتكاليف المسجلة والذمم المدينة.
- موازنة مشروع مبنية على WBS وأوزان إنجاز لا تُعتمد قبل اكتمال 100%.
- تحديثات تقدم مخطط/فعلي وتكلفة تراكمية بمسار اعتماد وتحديث مباشر لبطاقة المشروع.
- أوامر تغيير ذات أثر مالي وزمني، مرتبطة بسقوف الاعتماد وتحدّث العقد والمدة بعد الاعتماد فقط.
- مطالبات تأخير وتكلفة موثقة بتاريخ الحدث والإشعار، وسجل مخاطر وتوقعات تدفق نقدي شهرية.
- طلبات عروض ومقارنة فنية ومالية وترسية، مع مسار شراء مباشر مبرر.
- فواتير موردين ومطابقة ثلاثية إلزامية وقيود استحقاق وسداد متوازنة.
- صلاحيات مستقلة للمستخدمين على مستوى كل مستودع.
- دورة صيانة فعلية من بلاغ العطل والتشخيص والاعتماد إلى التنفيذ والفحص والإغلاق، مع إيقاف تحميل تشغيل المعدة أثناء الصيانة.
- طلبات فحص جودة ونتائج قبول/رفض تنشئ NCR تلقائيًا، وإجراءات تصحيحية لا تُغلق دون تحقق مستقل.
- تصاريح عمل وحوادث واجتماعات سلامة وأوامر إيقاف فوري لا تُستأنف دون توثيق إزالة الخطر.
- سجل وثائق بإصدارات واعتماد وتواريخ صلاحية، ومراسلات واردة وصادرة مرتبطة بالمشاريع.
- أستاذ عام فعلي بقيود يدوية متوازنة وترحيل غير قابل للتعديل وتصحيح بقيد عكسي فقط.
- فترات مالية تمنع الترحيل والعكس بعد الإقفال، مع ميزان مراجعة حي.
- إقرارات VAT محسوبة من فواتير العملاء والموردين، وتعديلات واعتماد مزدوج وتوثيق التقديم.
- التزامات ضريبة الاستقطاع وفحص جاهزية محلي لفواتير ZATCA؛ الإرسال الإنتاجي الخارجي غير مفعّل في هذا الإصدار.

## المتطلبات

- PHP 8.1 أو أحدث (يوصى 8.3)
- MySQL 8 أو أحدث
- إضافات PHP: `pdo`, `pdo_mysql`, `mbstring`, `fileinfo`, `json`
- Apache مع `mod_rewrite` و`mod_headers`، أو Nginx مكافئ
- HTTPS إلزامي في الإنتاج

## التركيب على cPanel

1. فك الضغط في مجلد خارج `public_html` إن أمكن، واجعل Document Root للنطاق يشير إلى مجلد `public` داخل النظام.
2. من cPanel أنشئ قاعدة MySQL ومستخدمًا، ثم اربط المستخدم بالقاعدة مع جميع الصلاحيات.
3. تأكد أن `storage` قابل للكتابة بواسطة PHP (`775` غالبًا)، ولا تستخدم `777` إلا مؤقتًا للتشخيص.
4. افتح `https://your-domain.example/install.php`.
5. أدخل بيانات قاعدة البيانات وحساب المدير العام. كلمة المرور لا تقل عن 10 أحرف.
6. سجّل الدخول وتأكد من ظهور حالة «محفوظ» أعلى الشاشة.
7. احذف `public/install.php` بعد نجاح الدخول. ملف القفل يمنع إعادة التثبيت، لكن حذف المثبّت أفضل.
8. اختبر `https://your-domain.example/health.php` ويجب أن يعيد `database: connected`.

## ترقية النسخة المثبتة إلى 1.10

1. خذ نسخة احتياطية من قاعدة البيانات ومجلد المشروع.
2. ارفع ملفات الإصدار 1.10 فوق ملفات الإصدار السابق مع إبقاء ملف `.env` ومجلد `storage/uploads` كما هما.
3. سجل الدخول بحساب المدير العام.
4. افتح `public/upgrade.php` من رابط النظام واضغط «تنفيذ الترقية الآن» مرة واحدة.
5. أضف إلى `.env`: `APP_URL=https://your-domain.example` و`MAIL_FROM_ADDRESS=noreply@your-domain.example`، ثم تأكد أن الاستضافة ترسل بريد PHP فعلًا.
6. من الموارد البشرية افتح ملف الموظف؛ سيظهر زر «تعديل البيانات» للحسابات التي تملك صلاحية `hr.employee.update`.
7. للموظفين الموجودين سابقًا: افتح «الأدوار والصلاحيات ← حساب دخول لموظف» وأدخل الهوية والبريد وكلمة المرور لكل موظف.
8. اختبر الدخول برقم الهوية، ثم اختبر رابط «نسيت كلمة المرور» من بريد حقيقي.
9. افتح `public/system-check.php` وتأكد أن حالات الفشل تساوي صفرًا وراجع تنبيهات الموظفين غير المهيئين.
10. احذف `public/upgrade.php` و`public/system-check.php` من السيرفر بعد نجاح الترقية.

الترقية لا تحذف بيانات الإصدار السابق، وتستخدم جدول `schema_migrations` لمنع تكرار التنفيذ.

إذا تعذر تغيير Document Root، انقل محتويات `public` فقط إلى `public_html`، ثم عدّل مسارات `dirname(__DIR__)` لتشير إلى مجلد النظام المحمي. الخيار الأول أكثر أمانًا.

## التشغيل عبر Docker

قبل الإنتاج، غيّر كلمتي مرور MySQL في `docker-compose.yml` والقيمة داخل healthcheck بحيث تطابق كلمة مرور root.

```bash
docker compose up -d --build
```

ثم افتح `http://localhost:8080/install.php` واستخدم:

- Host: `db`
- Port: `3306`
- Database: `sawaed_erp`
- User: `sawaed_user`
- Password: القيمة التي وضعتها في `docker-compose.yml`

## النسخ الاحتياطي

نفّذ نسخة يومية من قاعدة البيانات ومجلد `storage/uploads`. مثال MySQL:

```bash
mysqldump --single-transaction --routines --triggers -u sawaed_user -p sawaed_erp > sawaed_erp_$(date +%F).sql
```

احتفظ بنسخة مشفرة خارج الخادم، وطبّق اختبار استعادة شهري. النسخة الاحتياطية التي لم تُختبر استعادتها لا تُعد نسخة مضمونة.

## واجهات API

الجلسة نفسها المستخدمة في الويب تحمي الواجهات. في طلبات الكتابة أرسل رأس `X-CSRF-Token` بالقيمة الموجودة في `GET api/index.php?resource=me`.

```text
GET    api/index.php?resource=projects&limit=50&offset=0
GET    api/index.php?resource=projects&id=1
POST   api/index.php?resource=projects
PATCH  api/index.php?resource=projects&id=1
DELETE api/index.php?resource=equipment&id=1
GET    api/index.php?resource=audit
GET    api/users.php
POST   api/users.php
PATCH  api/users.php?id=2
POST   api/employee-onboarding.php
PATCH  api/employee-management.php
POST   api/upload.php
```

الموارد المتاحة: `projects`, `employees`, `equipment`, `inventory`, `suppliers`, `approvals`.

## ضوابط مهمة قبل الإطلاق المؤسسي

- استبدل البيانات التجريبية ببيانات الشركة عبر خطة النقل الموجودة في `DATA_MIGRATION_SPEC.md`.
- فعّل نسخًا احتياطيًا يوميًا، WAF، مراقبة أخطاء، وسياسة كلمات مرور ومصادقة متعددة العوامل على مستوى الاستضافة.
- لا تعتبر تكامل ZATCA أو البصمة أو WhatsApp أو البنك مفعّلًا لمجرد ظهور شاشته؛ هذه تحتاج مفاتيح ربط واعتماد واختبارات منفصلة.
- نفّذ UAT لكل إدارة، ثم تشغيلًا موازيًا ومطابقة مالية قبل اعتماد النظام كمصدر محاسبي وحيد.
- غيّر إعدادات `upload_max_filesize` و`post_max_size` في PHP إذا رغبت في حد مختلف للملفات.

## بنية المشروع

```text
app/                 التهيئة، الاتصال، الجلسات والحماية
database/            مخططات MySQL وSQLite
public/              جذر الويب، النظام، المثبّت وواجهات API
storage/uploads/     المستندات المحمية
storage/logs/        سجلات التطبيق المستقبلية
tools-closing-engine محرك الإقفال المالي واختباراته من النسخة السابقة
```

## بيانات الدخول

لا توجد كلمة مرور افتراضية. حساب المدير العام يُنشأ أثناء التثبيت، وهذا يمنع بقاء بيانات دخول عامة معروفة.
