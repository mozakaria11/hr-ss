# ملاحظات الإصدار 1.10.0-server

## الجديد في 1.10

- صلاحيات منفصلة لإنشاء الموظف وتعديل بياناته وإدارة حساب دخوله، ومُنحت افتراضيًا للمدير العام والمدير التنفيذي ومدير الموارد البشرية.
- شاشة تعديل شاملة لملف الموظف مع مزامنة الاسم والبريد في حساب الدخول والراتب الأساسي في العقد النشط.
- منع المستخدم العادي من قراءة ملفات زملائه أو رفع دوره إلى أدوار إدارية عبر واجهات API.
- وضع مخصص لتطبيق الموظف على الجوال، مع مشروع Android WebView للنطاق `ss.sawaedarab.com` يدعم الموقع والكاميرا والملفات والتنزيلات.
- فحص سلامة جديد للتأكد من وجود ترحيل 1.10 وصلاحيات الموارد البشرية الجديدة.

# ملاحظات الإصدار 1.9.0-server

## الجديد في 1.9

- تسجيل دخول الموظف برقم الهوية/الإقامة أو البريد مع الإبقاء على دخول الإدارة بالبريد.
- استعادة كلمة المرور بالبريد برمز مشفر أحادي الاستخدام ينتهي خلال 30 دقيقة ورسالة لا تكشف وجود الحساب.
- نموذج إضافة موظف موسع مستوحى من شاشة الموارد البشرية المرفقة، ويشمل الهوية والوظيفة والبريد وكلمة المرور والنوع والكفالة والصورة والجنسية والجوال والجنس والتواريخ واللغة والدور.
- إنشاء ملف الموظف وحساب المستخدم والربط والدور والعقد داخل معاملة واحدة مع التراجع الكامل عند الفشل.
- تهيئة حساب دخول للموظف الموجود مسبقًا مع إلزام الهوية والبريد، وفحوص سلامة توضح الحسابات غير الجاهزة.
- حماية الصور الشخصية خلف جلسة الدخول، وقفل مؤقت بعد تكرار محاولات الدخول الفاشلة.

# ملاحظات الإصدار 1.8.0-server

## الجديد في 1.8

- أستاذ عام خادمي وميزان مراجعة وقيود يدوية متوازنة.
- منع حذف أو تعديل القيود المرحلة؛ التصحيح بقيد عكسي مرتبط بالأصل.
- فترات مالية وإقفال يمنع أي ترحيل أو عكس داخل الفترة المغلقة.
- احتساب VAT من ضريبة المخرجات والمدخلات والتعديلات، مع اعتماد مزدوج وتوثيق التقديم.
- احتساب واعتماد وسداد ضريبة الاستقطاع.
- فحص جاهزية محلي لفواتير ZATCA مع بصمة SHA-256 وسجل نتائج؛ لا يوجد إرسال إنتاجي خارجي.
- تحديث تقارير الضرائب وحركة الأستاذ من البيانات الفعلية.

# ملاحظات الإصدار 1.7.0-server

# ملاحظات الإصدار 1.6.0-server

# ملاحظات الإصدار 1.5.0-server

## الجديد في 1.5

- موازنات مشاريع وإصدارات WBS وبنود تكلفة وأوزان تقدم.
- منع اعتماد الموازنة قبل وصول مجموع أوزان WBS إلى 100%.
- تحديثات تقدم مخطط وفعلي وتكلفة تراكمية بمسار اعتماد مستقل.
- منع تخفيض التقدم الفعلي عن آخر تحديث معتمد.
- أوامر تغيير بقيمة وأثر زمني وسقف اعتماد حسب الدور.
- تحديث قيمة العقد ومدة المشروع فقط بعد اعتماد أمر التغيير.
- مطالبات تأخير وتكلفة وتغيير نطاق مرتبطة بتاريخ الحدث والإشعار.
- توقعات تدفق نقدي شهرية قابلة للتحديث وسجل مخاطر لكل مشروع.
- ترقية تراكمية تنفذ جميع ترحيلات 1.1 حتى 1.5 الناقصة فقط.

## الجديد في 1.4

- عقود عملاء ومقاولي باطن مرتبطة بالمشروع والأطراف والقيم والمحتجز والدفعات المقدمة.
- مستخلصات تراكمية مع منع تجاوز قيمة العقد.
- احتساب آلي للمحتجز واسترداد الدفعة والخصومات وضريبة 15% وصافي الاستحقاق.
- اعتماد مستخلص العميل ثم إصدار فاتورة داخلية وقيد إيراد وضريبة متوازن.
- تحصيل جزئي أو كامل مع منع تجاوز الرصيد وإنشاء قيد بنك/ذمم.
- اعتماد مستخلص مقاول الباطن وإنشاء قيد استحقاق ومحتجز وضريبة مدخلات.
- تقرير ربحية حي يجمع الإيراد والتكاليف والرواتب والمواد والمقاولين والذمم.
- حالة ZATCA واضحة `not_submitted` حتى يتم ربط بيئة الاعتماد الخارجية فعليًا.

## الجديد في 1.3

- مستودعات متعددة وأرصدة منفصلة لكل مستودع وصنف.
- طلب شراء محفوظ واعتماد بالقيمة وفق سقوف الصلاحيات المعتمدة.
- تأهيل المورد شرط إلزامي قبل إصدار أمر الشراء.
- أمر شراء مرتبط بالطلب والمشروع والمورد مع منع تكرار التحويل.
- استلام وفحص جودة؛ المواد المرفوضة لا تدخل الرصيد.
- صرف مواد للمشاريع مع منع الرصيد السالب وسجل حركة دائم.
- سجل معدات ومحاضر نقل بين المشاريع، ولا يتغير مشروع المعدة قبل اعتماد النقل.
- ترقية تراكمية تنفذ ترحيلات 1.1 و1.2 و1.3 الناقصة فقط.

## الجديد في 1.2

- تسجيل حضور وانصراف فعلي من تطبيق الموظف باستخدام الكاميرا والموقع الجغرافي ونطاق المشروع.
- دعم ورديتي النهار والليل وفترة السماح، واحتساب دقائق التأخير تلقائيًا.
- لوحة متابعة مباشرة وإقفال الغياب مع استثناء الإجازات المعتمدة والراحة الأسبوعية يوم الجمعة.
- ربط حسابات الدخول بملفات الموظفين وعرض سجل الحضور وآخر قسيمة راتب مرحّلة.
- دورة مسير رواتب كاملة: احتساب، مراجعة، اعتماد، ترحيل وإقفال الفترة.
- إنشاء قيد محاسبي متوازن تلقائيًا عند ترحيل الرواتب.
- ترقية تراكمية آمنة؛ شاشة الترقية تنفذ ترحيلات 1.1 و1.2 الناقصة فقط.

> فحص وجود الكاميرا يعمل مباشرة. مطابقة هوية الوجه البيومترية تتطلب ربط مزود معتمد عبر `window.SawaedFaceVerifier` مع تحقق خادمي قبل الاعتماد المؤسسي.

## الجديد في 1.1

- نواة صلاحيات دائمة: أدوار، صلاحيات، تعيين أدوار الموظفين وسقوف الاعتماد.
- ملفات عقود الموظفين والراتب والبدلات وساعات العمل.
- طلبات الإجازات والنقل ومسار الاعتماد والرفض الفعلي.
- تنفيذ نقل الموظف المعتمد بين المشاريع وتحديث التوزيع التشغيلي.
- وثائق الموظفين والعهد والورديات وسجل الحضور كجداول تشغيلية.
- مراكز تكلفة ونطاقات مستخدمين للفروع والمشاريع.
- شاشة ترقية آمنة من الإصدار السابق عبر `public/upgrade.php`.

## ما كان موجودًا في 1.0

- تحويل النموذج الأحادي إلى تطبيق PHP/MySQL قابل للتثبيت.
- إضافة الدخول والجلسات والأدوار وحماية CSRF وسجل التدقيق.
- إضافة حفظ مركزي ومزامنة الجداول الأساسية وREST API.
- إضافة رفع المستندات المحمي ومُثبّت cPanel/VPS.
- إضافة Docker وSQLite للتجربة وMySQL للإنتاج.
- الإبقاء على تصميم سواعد ووحداته، وربط النماذج الأساسية بالحفظ الدائم.

التكاملات التي تتطلب جهات خارجية — ZATCA، واتساب، البنوك، البصمة والخرائط — تحتاج مفاتيح اعتماد وبيئة اختبار قبل تشغيلها الفعلي، ولا تتضمن الحزمة أي مفاتيح سرية افتراضية.
